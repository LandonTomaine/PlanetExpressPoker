# Competitor Research

- Treat GlossGenius as the primary replacement benchmark unless the user says otherwise.
- Treat GlossGenius as a benchmark, not a blueprint.
- Identify the user need a competitor solves, then decide whether this product should match it, simplify it, or improve it within MVP scope.
- Use competitor research to validate real user needs, not to import full competitor scope.
- Keep MVP boundaries explicit.
