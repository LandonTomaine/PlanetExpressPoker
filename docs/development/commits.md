# Commits

## Commit Scope

- Keep commits focused on one task or one small related set of changes.
- Do not include unrelated cleanup or formatting churn.
- Validate the changed area before committing when practical.

## Commit Messages

- Keep commit messages short, imperative, and specific.
- Prefer plain language over rigid ceremony.

Examples:

- `Add architecture overview for Supabase stack`
- `Scaffold Vite React app`
- `Add Husky pre-push quality gate`

## Git Usage

- Use shell-based Git commands so hooks run consistently.
- Do not amend commits unless explicitly requested.
- Prefer a new follow-up commit over rewriting history during normal local work.
